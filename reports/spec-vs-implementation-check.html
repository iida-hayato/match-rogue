仕様側の不整合

Reward Select の扱いが文書間で矛盾

02_core_loop_and_stage_flow と 05_screen_and_ui_spec には Reward Select が残っていますが、additional_spec_shop_reward_select_removed では廃止が明記されています。

現実装は追加仕様に寄っており、Reward Select は存在しません。仕様書のベース文書側を更新しないと、今後も同じ誤読が起きます。

実装が先行している点

Auto Drop Seal

ドロップ無限化、discard 非返却、自動盤面補充、山札枯渇時終了の一連挙動が実装済み。

回帰テスト

終了判定と補充条件はテストで固定されているため、仕様の意図はコード側ではかなり明確。

ショップ

レリック 2 枠など、仕様書の古い例示よりも実装のほうが進んでいます。

結論

  • ゲームロジックは大筋で仕様に沿っています。
  • 一番大きい差分は「Reward Select 廃止」が仕様文書間で統一されていないことです。
  • もう一つ大きい差分は初期デッキ枚数 50 と実装 300 の乖離です。
  • Auto Drop Seal は仕様未整備の新機能として扱い、別紙化するのが安全です。
