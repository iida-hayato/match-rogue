Auto Drop Seal

Endgame 判定パターン

このレポートは、`Move 0` と山札の残数、盤面の空き、`Auto Drop Seal` の有無がどう終了判定に影響するかを整理したものです。

基本ルール
通常は空き + 移動不可で終了
レリック有効時
空きがあれば自動補充を待つ
枯渇時
山札が尽きたら終了

判定テーブル

実際に迷いやすい条件を、`Move` / 山札 / 盤面空き / レリックの4軸で整理しています。

Move 山札 盤面に空き Auto Drop Seal 判定
> 0 あり / なし あり / なし あり / なし 継続
0 あり あり なし 終了しうる ただし通常は Drop が残っていれば続行
0 あり あり あり 継続 自動補充待ち
0 なし あり なし ゲームオーバー
0 なし あり あり ゲームオーバー
0 あり / なし なし なし 終了候補 Auto Drop の待ち対象ではない
0 あり / なし なし あり 終了候補 空きがないので自動補充は発生しない

判定の要点

通常時

Move が尽きていて、Drop も尽きていて、盤面に空きがあれば終了です。

Auto Drop Seal 時

Move が 0 で盤面に空きがあり、山札が残っている間だけ自動補充を待ちます。

山札枯渇時

自動補充も止まるので、空きが残っていればその時点で終了します。

実装参照

[scripts/ui/stage_play_screen.gd] - `resolve_board()` で `Auto Drop Seal` の自動補充を呼ぶ - `_should_end_run()` で終了条件を分岐 - `_on_drop_pressed()` で手動 Drop を封じる [scripts/domain/cascade_resolver.gd] - `refill_from_deck()` が `relic_auto_drop_seal` を見て山札の戻しを止める [scripts/domain/deck_state.gd] - `draw_one(false)` はリシャッフルを行わず `null` を返す

フォローアップ

  • この判定は「空きがあるか」と「山札が残っているか」をセットで見るのが重要です。
  • レリックのない通常ルートに副作用が出ないよう、回帰テストで分岐を固定しています。
